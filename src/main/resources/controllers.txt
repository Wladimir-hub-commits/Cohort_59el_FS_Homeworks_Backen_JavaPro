выполнение одинаковой операции даёт тот же результат, что
 и первое выполнение, без изменения состояния системы после
  первого вызова.
Идемпотентность: да, повтор даёт один и тот же результат (повторение запроса меняет ли состояние)
Кэшируемость: да
Кэшируемость (Cacheability) в Java — это характеристика,
обозначающая возможность хранения и повторного использования
результатов выполнения методов или данных в кэше для повышения
производительности приложения. Когда результат метода считается
 кэшируемым, его можно сохранить в кэше, чтобы при повторных вызовах
  с одинаковыми параметрами возвращать уже вычисленный результат без
  повторных затрат времени и ресурсов.

Когда не нужно использовать GET запросы.
Не использовать: для создания или изменения данных
GET нужен только для получения данных
//////////////////////////////////////////
Post Запрос: чтобы что-то создать (новый ресурс - может быть пользователь или товар)
Например: для регистрации на каком то Веб-сайте, для создания нового пользователя сайта
Тело запроса: JSON - В Java "Джейсон" (JSON) — это формат обмена данными,
основанный на JavaScript Object Notation. Он используется для хранения и
передачи структурированных данных в виде текста, который легко читается
человеком и парсится программами.
Safe: нет
Идемпотентность: нет, повтор запроса создаст дупликат.(повторение запроса меняет ли состояние)
Кэшируемость: нет
Не использовать: получить данные
/////////////////////////////////////////////////
PUT Для чего: полная замена ресурса
Тело запроса: JSON
Safe: нет
Идемпотентность: да (повторение запроса меняет ли состояние)
Кэшируемость: нет
Не использовать: если меняется только часть полей или нет
возможности отправить объект целиком
PUT - использовать только если мы полностью меняем наш ресурс
PUT работает только тогда когда мы можем полный набор данных для замены
///////////////////////////////////////////////////
PATCH: Используется для частичного обновления (хорош когда я хочу поменять статус)
Тело запроса: JSON
Safe: нет
Идемпотентность: да (повторение запроса меняет ли состояние)
Кэшируемость: нет
///////////////////////////////////////////////////
DELETE: Для удаления ресурса (Если DELETE удаляется  правильно то данные
удаляются безвазвратно)
Тело запроса: нет
Safe: нет
Идемпотентность: да (повторение запроса меняет ли состояние)
Кэшируемость: нет
Не использовать: если запрещено физическое удаление данных
В некоторых Странах запрещено мягкое удаление
Данные о пользователе хранятся 10 лет

Ответы на Запросы
HTTP - статусы:
200 ок успешное чтение или обновление ресурса
2001 Created успешное создание ресурса (Post)
2004 No Content успешное обновление без тела
400 Bad Request невалидные данные (валидация DTO-data
 transfer object)
404 Not Found ресурс не найден
409 Conflict конфликт состояний
500 Internal Server Error - неожиданная ошибка
